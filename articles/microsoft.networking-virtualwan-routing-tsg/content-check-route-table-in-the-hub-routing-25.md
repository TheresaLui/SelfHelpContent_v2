# Check Route table in the HUB Routing

In this step you'll be checking if the branch and VNET connections for source and destination are associated and propagated to the same route table. Please use the steps below to do so:

In ASC HUB, navigate to the route table section and see if you can find the VNET connections and Branch connection of source and destination, associated and propagated to same route table. (Can be default route table or any custom route table customer created and wants to use)
