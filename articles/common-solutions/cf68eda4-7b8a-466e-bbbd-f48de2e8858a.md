# Peering session not established

BGP Session state on a peering connection is not in Established state. New Peering provision happens during the weekdays after peering connection is approved by Microsoft Peering Team, and notification is sent to registered email on peer ASN.


## **Recommended Steps**

1. Please see the latest notification regarding peering update from peer8075@microsoft.com
2. Please make sure configuration is completed on your end for BGP session to get established
3. If connection is in provisioning failed state, please see error message for details
