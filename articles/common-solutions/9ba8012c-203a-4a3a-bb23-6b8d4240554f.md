# SSAS exceptions, unexpected termination, not responding and unable to start

Most customers can resolve issues  with SQL Server Analysis Services exception, unexpected termination and non-responsiveness using following steps.

## **Recommended Steps**


- **SQL Server Analysis Services instance generated exception memory dump**
    SQL Server Analysis Services generates dump files capture data used for analyzing extraordinary events such as in response to a server crash, exception and configuration errors. For Best Practices, please consider upgrade to latest service pack and cumulative update with the [Determine the version, edition, and update level of SQL server and its components document](https://docs.microsoft.com/troubleshoot/sql/general/determine-version-edition-update-level).

- **SQL Server Analysis Services unexpectedly terminates**
    This can happen if the SQL Server Analysis Services instance encountered exception or error, you should review the [log](https://docs.microsoft.com/analysis-services/instances/log-operations-in-analysis-services?view=asallproducts-allversions&viewFallbackFrom=asallproducts-allversion) and application event log, and take proper action recommended by the article.

- **SQL Server Analysis Services instance becomes non-responding**
    This can happen if the SSAS encounter issue per  [SQL Server Analysis Services stops accepting new connections â€“ processing commit locks hurt](https://blogs.msdn.microsoft.com/jason_howell/2012/07/03/analysis-services-stops-accepting-new-connections-processing-commit-locks-hurt/), please review and take the suggested action.

- **SQL Server Analysis Services instance stop in the Azure VM and cannot be started up**
    This can happen if the SQL Server Analysis Services instance encounters exception or error during startup, you should review the [log](https://docs.microsoft.com/analysis-services/instances/log-operations-in-analysis-services?view=asallproducts-allversions&viewFallbackFrom=asallproducts-allversion) and application event log, and take proper action recommended by the article.

- Please run the [SSASDiag Data Collection Tool](https://github.com/ssasdiag/SSASDiag/releases) to collect data on the Analysis Services server when problem occurs. The info can be used for cause analysis.
