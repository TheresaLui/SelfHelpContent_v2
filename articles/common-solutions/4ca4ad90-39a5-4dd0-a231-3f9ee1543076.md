# Cannot find a SQL Image on Azure Portal 

Most users able to resolve their issues finding SQL images, and provisioning and deploying SQL VMs, by following the steps below. 
 
## **Recommended Steps** 

* **Cannot find the SQL image on Azure Portal**  

  There are many images that don't appear on Azure Portal. We show only the latest versions on the portal. Use the following script to look for the SQL version (`SQLver`) you want to deploy: 

  ```PS 
  Install-Module AzureRM -AllowClobber 
  Login-AzureRmAccount 
  $X = @() 
  $SQLVer = "SQL2016" 
  $locals = Get-AzureRmLocation 
  foreach($local in $locals) 
  { 
  $X += Get-AzureRmVMImageOffer -Publisher 'MicrosoftSQLServer' -Location $local.DisplayName | Get-AzureRmVMImageSku | Where {($_.Offer).Contains($SQLVer)}  
  } 
  $X | Get-AzureRMVMImage | Out-GridView -Title "Images" -PassThru 
  ``` 

* **Deploy SQL images with PowerShell** 

  You can easily [deploy SQL Images with PowerShell](https://docs.microsoft.com/azure/azure-sql/virtual-machines/windows/create-sql-vm-powershell) if you don't see the images on the portal. 

* **Cannot find SQL Express or Developer Version image on Azure Portal** 
   
  [SQL 2008/2008 R2, SQL 2012 and SQL 2014 developer images](https://docs.microsoft.com/azure/azure-sql/virtual-machines/windows/sql-server-on-azure-vm-iaas-what-is-overview#payasyougo) are not available on Azure portal. To work around this issue, create a Windows-only image and manually install the SQL Developer version on it. Install the [SQL IaaS Extension](https://docs.microsoft.com/azure/azure-sql/virtual-machines/windows/sql-agent-extension-manually-register-single-vm?tabs=bash%2Cazure-cli) and [change the licensing](https://docs.microsoft.com/azure/azure-sql/virtual-machines/windows/licensing-model-azure-hybrid-benefit-ahb-change?tabs=azure-portal) from the Azure portal, if required for compliance. 

  To deploy SQL Express images, create a Windows-only image and manually install the specific version of SQL Express on it. Install the [SQL IaaS Extension](https://docs.microsoft.com/azure/azure-sql/virtual-machines/windows/sql-agent-extension-manually-register-single-vm?tabs=bash%2Cazure-cli) and [change the licensing](https://docs.microsoft.com/azure/azure-sql/virtual-machines/windows/licensing-model-azure-hybrid-benefit-ahb-change?tabs=azure-portal) from the Azure portal, if required for compliance. 

* **Cannot deploy SQL VM due to capacity or quota limits** 

  Follow the instructions in [Increase the limits](https://docs.microsoft.com/azure/azure-portal/supportability/per-vm-quota-requests). 

* **Cannot Find SQL Always On templates on the Azure portal**  

  You can deploy the Always On setup by using the [Quick Start Template](https://docs.microsoft.com/azure/azure-sql/virtual-machines/windows/availability-group-quickstart-template-configure).
