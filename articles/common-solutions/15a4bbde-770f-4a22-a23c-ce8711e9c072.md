# I can't configure my Workspace

SQL insights stores it's monitoring data in a Log Analytics workspace.  You will need the proper access to this workspace to enable monitoring and view the monitoring data.


## **Recommended Steps**

1. When enabling SQL insights you need the [Log Analytics contributor role](https://docs.microsoft.com/azure/azure-monitor/logs/manage-access#manage-access-using-azure-permissions) on the workspace you will use.
2. When viewing monitoring data in a SQL insights monitoring profile you need the [Log Analytics reader role](https://docs.microsoft.com/azure/azure-monitor/logs/manage-access#manage-access-using-azure-permissions) on the workspace you will use.  This role does not allow for creating Alert rules, it only allows for viewing the monitoring data.
3. Verify that the workspace is in a [supported region](https://docs.microsoft.com/azure/azure-monitor/insights/sql-insights-enable) for SQL insights.
