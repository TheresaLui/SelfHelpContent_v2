# Issues signing into Microsoft applications

## **Recommended Steps**

Resolve some of the most common issues with signing in to Microsoft applications by using the following information.

**I am having issues signing into a Microsoft application**

• [I can’t sign in to a Microsoft application](https://docs.microsoft.com/azure/active-directory/manage-apps/application-sign-in-problem-first-party-microsoft?WT.mc_id=UI_AAD_Apps_Sign_In_Support_L2_MicrosoftApp)
• [Windows Hello for Business Deployment - known issues](https://docs.microsoft.com/windows/security/identity-protection/hello-for-business/hello-deployment-issues#identifying-server-2019-key-trust-authentication-issue)

**I am having other issues while signing into an application**

• [I can’t access ab application as a guest user](https://docs.microsoft.com/azure/active-directory/external-identities/b2b-quickstart-add-guest-users-portal)
• [I get error message, "You can’t get there from here", while trying to access my organization resources](https://docs.microsoft.com/azure/active-directory/user-help/user-help-device-remediation)
• [I can’t access the application because user assignment is missing for the application](https://docs.microsoft.com/azure/active-directory/manage-apps/assign-user-or-group-access-portal)
• [I can’t sign in to an application because access to tenant is denied](https://docs.microsoft.com/azure/active-directory/manage-apps/tenant-restrictions)
• I get the [AADSTS50000: Token Issuance Error](https://docs.microsoft.com/azure/active-directory/develop/reference-aadsts-error-codes?WT.mc_id=Portal-Microsoft_Azure_Support) and need to be assigned [Application admin role](https://docs.microsoft.com/azure/active-directory/roles/permissions-reference)
• [I am unable to login using service principal](https://docs.microsoft.com/cli/azure/create-an-azure-service-principal-azure-cli#manage-service-principal-roles)
• Resolve [AADSTS errors](https://docs.microsoft.com/azure/active-directory/develop/reference-aadsts-error-codes?WT.mc_id=Portal-Microsoft_Azure_Support) & [Troubleshoot signing in to SAML based SSO configured apps](https://docs.microsoft.com/azure/active-directory/manage-apps/application-sign-in-problem-federated-sso-gallery?WT.mc_id=UI_AAD_Apps_Sign_In_Support_L2_Gallery)
• [I get an error to verify my app's publisher domain](https://docs.microsoft.com/azure/active-directory/develop/howto-configure-publisher-domain#to-verify-a-new-domain-for-your-app) 
• I want to [configure Admin consent workflow](https://docs.microsoft.com/azure/active-directory/manage-apps/configure-admin-consent-workflow) and [Grant tenant-wide admin consent](https://docs.microsoft.com/azure/active-directory/manage-apps/grant-admin-consent#:~:text=The%20Global%20Administrator%20role%20is,to%20the%20Microsoft%20Graph%20API.&text=before%20granting%20consent.-,The%20Global%20Administrator%20role%20is%20required%20in%20order%20to%20provide,to%20the%20Microsoft%20Graph%20API)
• [I want to assign owner or restrict who can create/consent to applications](https://docs.microsoft.com/azure/active-directory/roles/delegate-app-roles)
• [I’m having trouble signing in to application(s) using Chrome browser only](https://docs.microsoft.com/office365/troubleshoot/miscellaneous/chrome-behavior-affects-applications?WT.mc_id=Portal-Microsoft_Azure_Support)
