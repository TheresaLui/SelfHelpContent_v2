# Edge Compute Modules.

Use the **Compute** page in the local web UI to assign IPs to each IoT Edge module or services running on your device that you want to access from the external network. After the IPs are assigned and you've deployed the module or app, there are multiple ways to figure out the IP assigned to each module, which you can use to access the module on the external network. 

Get the external IP address assigned to IoT Edge modules or services running on Azure Stack Edge Pro device.

## **Recommended Steps**

1.	**Via `iotedge` commands**: Connect to the PowerShell interface of the device. You can then use the iotedge list command to view all the IoT Edge modules or services running on your device. The sample output will also indicate the IP assigned to each module, where it can be accessed from the external network. 
2. **Via the Kubernetes dashboard**: Use the Kubernetes dashboard and filter all the services that are running in the `iotedge` namespace on your Azure Stack Edge Pro device to see their corresponding external IPs. 
3. **Via `kubectl`**: Use the `kubectl` commands to get the external IP address of a service running in the `iot edge` namespace:
```
.\kubectl.exe get svc -n iotedge
```
