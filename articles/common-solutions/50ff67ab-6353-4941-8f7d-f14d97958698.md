# Kubernetes How To

On your Azure Stack Edge Pro device, a Kubernetes cluster is created when you configure a compute role. After the Kubernetes cluster is created, you can deploy containerized applications in pods. 

You can connect to and manage the cluster locally from a client machine through a native tool, such as `kubectl`. If you face issues changing the IP range of the internal IP address used by Kubernetes, review the recommended steps.

To change the memory and processors allocated to a Kubernetes worker node, use the `Get-AzureDataBoxEdgeRole` and `Set-AzureDataBoxEdgeRoleCompute` cmdlets through the PowerShell interface of your Azure Stack Edge Pro device.

## **Recommended Steps**

1. [Create a Kubernetes cluster](https://docs.microsoft.com/azure/databox-online/azure-stack-edge-gpu-create-kubernetes-cluster)
2. [Workload management with Kubernetes on your device](https://docs.microsoft.com/azure/databox-online/azure-stack-edge-gpu-kubernetes-workload-management)
3. [Troubleshoot Kubernetes performance issues](https://docs.microsoft.com/azure/databox-online/azure-stack-edge-gpu-monitor-kubernetes-dashboard)
4. [Change Kubernetes pod and service subnets](https://docs.microsoft.com/azure/databox-online/azure-stack-edge-gpu-connect-powershell-interface#change-kubernetes-pod-and-service-subnets)
5. [Change memory, processor limits for Kubernetes worker node](https://docs.microsoft.com/azure/databox-online/azure-stack-edge-gpu-connect-powershell-interface#change-memory-processor-limits-for-kubernetes-worker-node)
