# Kubernetes How To

On your Azure Stack Edge device, a Kubernetes cluster is created when you configure compute role. After the Kubernetes cluster is created, containerized applications can be deployed on the Kubernetes cluster in pods. There are specific ways to deploy workloads in your Kubernetes cluster.

After the Kubernetes cluster is created, connect to and manage the cluster locally from a client machine through a native tool, such as `kubectl`. If you encounter issues changing the IP range of the internal IP address used by Kubernetes, review the following steps.

If you want to change the memory and processors allocated to a Kubernetes worker node, use the `Get-AzureDataBoxEdgeRole` and `Set-AzureDataBoxEdgeRoleCompute` cmdlets through the PowerShell interface of your Azure Stack Edge Pro device.
