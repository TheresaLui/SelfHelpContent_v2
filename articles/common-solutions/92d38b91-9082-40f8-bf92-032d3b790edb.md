# Azure SQL MI network settings

SQL insights will need access to your Azue SQL managed instance in order to collect monitoring data.  We recommend using a private endpoint, but may also make firewall settings to allow access for monitoring purposes.

## **Recommended Steps**

1. Please review our [documentation for setting up your Azure SQL managed instance](https://docs.microsoft.com/azure/azure-monitor/insights/sql-insights-enable#azure-sql-managed-instances) so that it can be accessed by the monitoring VM.
2. For setting up access when the monitoring VM is in the same VNET, [refer to this document](https://docs.microsoft.com/azure/azure-sql/managed-instance/connect-application-instance#connect-inside-the-same-vnet).
3. For setting up access when the monitoring VM is in a different VNET, [refer to this document](https://docs.microsoft.com/azure/azure-sql/managed-instance/connect-application-instance#connect-inside-a-different-vnet).
