# Kubernetes Storage

On the Azure Stack Edge device, statically provisioned PersistentVolumes are created by using the device's storage capabilities. When you provision a share and use the share with Edge compute option enabled, this action creates a PV resource automatically in the Kubernetes cluster. You can create both SMB and NFS shares to statically provision PVs on Azure Stack Edge device. After the PV is provisioned, submit a PVC to claim this storage.

## **Recommended Steps**

1.	Define the storage class
2.	Claim the storage
3.	Provision the storage dynamically
4.	Mount the PVC to the container

## **Recommended Documents**

* [Kubernetes Storage]( https://docs.microsoft.com/azure/databox-online/azure-stack-edge-gpu-kubernetes-storage#staticprovisioning)

### Notes
* Kubernetes cluster admins can statically provision the storage. To do so, they can use storage backend based on SMB/NFS file system, or use iSCSI disks that attach locally over the network in an on-premises environment. They can also use Azure Files or Azure Disks in the cloud.
