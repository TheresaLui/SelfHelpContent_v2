# How-To/Portal and Client Tools for SQL on-demand

## **Recommended Steps**

* To obtain connection string for SQL on-demand endpoint, check [Connect to SQL Analytics](https://review.docs.microsoft.com/azure/synapse-analytics/sql-analytics/connect-overview?toc=/azure/synapse-analytics/toc.json&bc=/azure/synapse-analytics/breadcrumb/toc.json&branch=release-ignite-arcadia).

* To connect and query SQL on-demand using Azure Data Studio, follow [steps outlined here](https://review.docs.microsoft.com/azure/synapse-analytics/sql-analytics/get-started-azure-data-studio?toc=/azure/synapse-analytics/toc.json&bc=/azure/synapse-analytics/breadcrumb/toc.json&branch=release-ignite-arcadia). Make sure your network does not block port 1433.

* To connect and query SQL on-demand using SSMS, follow [steps outlined here](https://review.docs.microsoft.com/azure/synapse-analytics/sql-analytics/get-started-ssms?toc=/azure/synapse-analytics/toc.json&bc=/azure/synapse-analytics/breadcrumb/toc.json&branch=release-ignite-arcadia). Make sure your network does not block port 1433.

* To connect Power BI to SQL on-demand, follow [steps outlined here](https://review.docs.microsoft.com/azure/synapse-analytics/sql-analytics/get-started-power-bi-professional?toc=/azure/synapse-analytics/toc.json&bc=/azure/synapse-analytics/breadcrumb/toc.json&branch=release-ignite-arcadia). Make sure your network does not block port 1433.

* If using Synapse Studio, make sure your network does not block port 1443.
