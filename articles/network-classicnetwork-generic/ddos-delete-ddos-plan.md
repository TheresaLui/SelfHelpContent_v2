# General guidance or advisory

## **Recommended Steps**

* Manage Azure DDoS Protection using the Azure [portal](https://docs.microsoft.com/azure/virtual-network/ddos-protection-manage-portal)
* Review the Azure DDoS Protection [offerings](https://azure.microsoft.com/blog/azure-ddos-protection-for-virtual-networks-generally-available/), [best practices](https://docs.microsoft.com/azure/security/azure-ddos-best-practices), and a service [overview](https://docs.microsoft.com/azure/virtual-network/ddos-protection-overview)
* Read about how to [Disable DDoS Protection](https://docs.microsoft.com/azure/virtual-network/manage-ddos-protection#disable-ddos-for-a-virtual-network) for all associated Virtual Networks
* Read more about  [deleting a DDoS Plan resource](https://docs.microsoft.com/azure/virtual-network/manage-ddos-protection#work-with-ddos-protection-plans)

Please Note: Exclusion of individual public IPs on a VNet is currently not supported.
