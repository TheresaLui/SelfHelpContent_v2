# Connectivity to Microsoft Public Services

You can use below documents to learn how to plan network implementation project for deploying ExpressRoute for Microsoft Public Services (SaaS and PaaS) such as Office 365, Azure Storage, Azure Web Apps, Dynamics 365, and more.

## **Recommended Documents**

* [ExpressRoute for Office 365 Overview](https://docs.microsoft.com/office365/enterprise/azure-expressroute) 
* [ExpressRoute for Office 365 Implementation](https://docs.microsoft.com/office365/enterprise/implementing-expressroute)
* [Network planning with ExpressRoute](https://docs.microsoft.com/office365/enterprise/network-planning-with-expressroute) 
* [Routing](https://docs.microsoft.com/office365/enterprise/routing-with-expressroute) with ExpressRoute for Office 365
* [Route filters](https://docs.microsoft.com/azure/expressroute/how-to-routefilter-portal) need to be configured to consume a subset of services through Microsoft Peering
* Find answers to [frequently asked questions](https://docs.microsoft.com/azure/expressroute/expressroute-faqs#expressroute-for-office-365) on ExpressRoute
