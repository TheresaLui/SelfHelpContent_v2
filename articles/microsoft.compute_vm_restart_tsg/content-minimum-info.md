# How to check for minimum information required to start the case

Review the customer verbatim for the following and validate that the case has all of the minimum required details shown below to start working:
   1. Clear issue description
   2. Subscription ID
   3. VM name
   4. Restart Date and Time Frame.
   5. Workloads/software/applications/Role being hosted on the VM (if possible)
   6. Business Impact: "How would one describe the business impact of this issue?"
   7. Approval to collect the logs. If the case is submitted through the Portal, you would see "- GrantPermission: true" in the Case Description. 

**Note:** Customer sharing of credentials, storage keys, or publish settings files are not allowed. Under no circumstances should you be requesting this information. If the customer provides credentials, storage keys, or publish settings files please contact the customer with the appropriate [template](https://microsoft.sharepoint.com/teams/WATS/_layouts/OneNote.aspx?id=%2Fteams%2FWATS%2FSiteAssets%2FWATS%20Notebook&wd=target%281_SOP%2FDPP.one%7cA74770A7-52A4-4FC2-83A3-DFEC3ABCA630%2FTemplate%3a%20Response%20to%20Customers%20Giving%20Us%20Credentials%7c9FF0E60C-C3DE-44E0-8B7D-B84E7CB0031C%2F%29) and reach out to your area's [Data Protection Manager](https://microsoft.sharepoint.com/teams/dataprotectionipg/Pages/Data-Protection-Leads.aspx) with the case number as the information needs to be purged from MSSolve to avoid a data breach. Please refer to the [Data Classification](https://microsoft.sharepoint.com/teams/dataprotectionipg/Pages/Services-Data-Classification.aspx) and [Data Storage](https://microsoft.sharepoint.com/teams/dataprotectionipg/Pages/Services-Data-Storage.aspx) policies for further information'.
