# Azure Stack Node Management issues

Using the Azure Stack Administrator portal, you can view each physical node's status, and run node actions such as power on, power off, shut down, drain, resume, and repair. Most support questions about node management are covered in the following documents. 

## **Recommended Steps**

- [Azure Stack is showing wrong node status](https://github.com/troettinger/TomTools/blob/master/articles/azurestack-operator-nodestatus.md)
- Follow steps to [perform Scale-unit actions](https://docs.microsoft.com/azure/azure-stack/azure-stack-node-actions#scale-unit-node-actions) to start, stop, drain, resume, repair, or shutdown a node
- If needed, you can [add memory](https://docs.microsoft.com/azure-stack/operator/azure-stack-manage-storage-physical-memory-capacity) or [replace a scale unit node](https://docs.microsoft.com/azure/azure-stack/azure-stack-replace-node)


## **Recommended Documents**

- [Scale unit node actions in Azure Stack](https://docs.microsoft.com/azure/azure-stack/azure-stack-node-actions)
- [Replace a scale unit node on an Azure Stack integrated system](https://docs.microsoft.com/azure/azure-stack/azure-stack-replace-node)
- [How to view the node status](https://docs.microsoft.com/azure/azure-stack/azure-stack-node-actions#view-the-node-status)
