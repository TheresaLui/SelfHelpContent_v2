# Issues Deploying Virtual Machines in Azure Stack

Azure Stack offers you virtual machines (VMs) as one type of an on-demand, scalable computing resource. You can choose a VM when you need more control over the computing environment than the other choices.

## **Recommended Steps**

1. Review [design considerations for your VM](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-compute-overview#what-do-i-need-to-think-about-before-creating-a-vm), and understand [differences between VM features on Azure Stack](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-vm-considerations) and Microsoft Azure
2. Review [virtual machine sizes supported in Azure Stack](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-vm-sizes)
3. Create your VM with using the desired operating system, using the interface of your choice:

    - Create a Windows virtual machine with the [Azure Stack portal](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-quick-windows-portal), [PowerShell for Azure Stack](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-quick-create-vm-windows-powershell) or using [Azure CLI](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-quick-create-vm-windows-cli)
    - Or create a Linux virtual machine by using the [Azure Stack portal](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-quick-linux-portal),  [PowerShell in Azure Stack](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-quick-create-vm-linux-powershell), or using [Azure CLI](https://docs.microsoft.com/azure/azure-stack/user/azure-stack-quick-create-vm-linux-cli)
