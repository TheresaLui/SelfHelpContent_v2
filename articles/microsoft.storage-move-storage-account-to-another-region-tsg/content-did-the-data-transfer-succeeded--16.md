# Check if the data transfer succeeded

1. Upload, download, and manage Azure blobs, files, queues, and tables, as well as Azure Cosmos DB and Azure Data Lake Storage entities. Easily access virtual machine disks, and work with either Azure Resource Manager or classic storage accounts. Manage and configure cross-origin resource sharing rules. 
2. Download Storage Explorer [here](https://azure.microsoft.com/en-us/features/storage-explorer/) 
3. Connect your account 
    1. In Storage Explorer, select View  Account Management or select the Manage Accounts button. ACCOUNT MANAGEMENT now displays all the Azure accounts you've signed in to. To connect to another account, select Add an account. In Connect to Azure Storage, select an Azure cloud from Azure environment to sign in to a national cloud or an Azure Stack. After you choose your environment, select Next. 
4. Copy data: 
    1. In the left pane, expand the storage account containing the blob container you wish to manage. Select the storage account and display the blob containers, file shares, tables or queues. There is no way to copy/paste a storage account with all it's contents from one SA to another. You will have to select the blob container, or file share and copy paste each one from source to destiny storage account.
