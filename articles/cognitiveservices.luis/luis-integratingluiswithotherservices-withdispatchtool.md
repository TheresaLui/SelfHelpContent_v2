# With Dispatch Tool

## **Recommended Steps**

* [Dispatch is a CLI tool](https://docs.microsoft.com/azure/bot-service/bot-builder-tutorial-dispatch?view=azure-bot-service-4.0&tabs=cs) that allows you to use LUIS and [QnA Maker](https://docs.microsoft.com/azure/cognitive-services/qnamaker/) in one application

* Dispatch allows you to create and evaluate LUIS models used to dispatch intent across multiple bot modules, such as LUIS models, QnA knowledge bases and others

* To properly use dispatch app, you need to give it the [proper read access permissions to existing apps](https://docs.microsoft.com/azure/bot-service/bot-builder-tutorial-dispatch?view=azure-bot-service-4.0&tabs=cs#dispatch-app-needs-read-access-to-existing-apps)

* Learn how you can [test and evaluate your Dispatch model](https://docs.microsoft.com/azure/bot-service/bot-builder-tutorial-dispatch?view=azure-bot-service-4.0&tabs=cs#resolving-incorrect-top-intent-from-dispatch)

* Learn how you can [install Dispatch tool](https://github.com/microsoft/botbuilder-tools/tree/master/packages/Dispatch)

* Understand [when to LUIS and when to use QnA and how these two services compliment each other](https://docs.microsoft.com/azure/cognitive-services/luis/choose-natural-language-processing-service#when-to-use-each-service)

* Implement and learn more about the [best practices](https://docs.microsoft.com/azure/cognitive-services/luis/choose-natural-language-processing-service#best-practices) for each service
