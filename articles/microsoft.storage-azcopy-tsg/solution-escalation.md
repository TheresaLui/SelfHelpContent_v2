# Escalation

<!--issueDescription-->
1. Leverage the **Ava** bot on the **Storage SME Channel**
2. Target the **AzCopy CSS-PG discussion** meeting at **2:30 PM EDT Tuesdays**
3. ICM **(requires TA approval)**

**Please include the following details in ICM apart from what is already being asked in the template. More importantly, consult your TA, all storage CRIs require approval.**

1. Which version of the AzCopy was used?
    1. Note: The version is visible when running AzCopy without any argument
2. Which platform are you using? (ex: Windows, Mac, Linux)
3. What command did you run?
    1. Note: Please remove the SAS to avoid exposing your credentials. If you cannot remember the exact command, please retrieve it from the beginning of the log file.
4. What problem was encountered?
5. How can we reproduce the problem in the simplest way?
6. Have you found a mitigation/solution?
7. For **Severity B and C** issues, you should engage the EEE team, and provide the appropriate template. **If you can't create the ICM in ASC** use this link:
   1. Storage: [**CRI**](http://aka.ms/cri-xeee)
8. For **Severity A and 0** issues, you should engage WASU, and provide the appropriate template. **If you can't create the ICM in ASC** use this link:
   1. Storage Resource Provider: [**CRI**](http://aka.ms/CXPCRI-StorageSRP)

<!--/issueDescription-->
